【システム概要】
回答一致率を出すようなシステムを作成する。
解答を先に集める必要があるので、協力を依頼することとする。



【ディレクトリ構成】
/01.document
  ・デザイン（psdファイル）

/02.cording
  ・ソース一式



【前提】
・質問や人の回答はjsonで保持。
・自分の選択した結果と一致度の高い人のランキングが分かれば良い。
・レスポンシブ対応は適当。PC版もデザイン通りといかなくても良い



【こうなった】
・10問だと結果があまりばらつかない印象だったので、15問とした。
・結果の一致率が50%以上だったら、上位だと思う。
・自分を結果から外す場合は、URLの最後に「?sasamoto」のようにパラメータをつけるようにした。
・最初のページでoptionボタンを押下しながらstartボタンを押下することで、コンソールに自分と回答が一致した人の名前を出すようなログモードを用意した。
・質問が右から出てきて左に消えていくが、CSSで対応している。HTML的には全て残っている。
・結果一覧ページも用意した。
・結果一覧ページの円グラフはsvgを作ることで対応した。結構気に入っている。
・アンケートページとして、データ収集用ページを用意した。その結果のidとnameだけを書き換えてjsonに貼り付ければシステムに反映されるようにした。



【jsonの仕様】
main.json       質問一覧を保持
{
  "theme": テーマ名を指定する
  "questions": [  問題は配列で保持する！！
    {
      "qid": 質問IDを指定する（内部的には使用しなかったので、不要であると言える）
      "text": 質問文を指定する
      "type": 常に"radio"とすること
      "choices":[ 選択肢は配列で保持する！！
        {
          "text": 選択肢のテキスト
          "code": 選択したテキストを示すユニークID
        },
        ・・・
      ]
    },
    ・・・
  ]
}

person.json     個人の結果を保持
{
  "person": [
    {
      "id": 人を示すID（画像ファイル名や自分を除外するURLのパラメータに使用する）
      "name": 表示する名前
      "results": 結果一覧。右の様なファオーマット[{"qid": "xx", "ans": "yy"}, ・・・]
    },
    ・・・
  ]
}
